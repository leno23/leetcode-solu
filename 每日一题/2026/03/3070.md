3070. 元素和小于等于 k 的子矩阵的数目

给你一个下标从 0 开始的整数矩阵 grid 和一个整数 k。
返回包含 grid 左上角元素、元素和小于或等于 k 的 子矩阵的数目。

**示例 1：**
**输入：** grid = [[7,6,3],[6,6,1]], k = 18
**输出：** 4
**解释：** 只有 4 个子矩阵满足：包含 grid 的左上角元素，并且元素和小于或等于 18 。

**示例 2：**
**输入：** grid = [[7,2,9],[1,5,0],[2,6,6]], k = 20
**输出：** 6
**解释：** 只有 6 个子矩阵满足：包含 grid 的左上角元素，并且元素和小于或等于 20 。

**提示：**
* m == grid.length 
* n == grid[i].length
* 1 <= n, m <= 1000 
* 0 <= grid[i][j] <= 1000
* 1 <= k <= 10^9

---

### 解题思路：二维前缀和

题目要求找出所有包含左上角元素且元素和小于等于 `k` 的子矩阵的数目。
由于所有元素都是非负整数（`0 <= grid[i][j] <= 1000`），因此随着子矩阵的范围扩大（向右或向下扩展），子矩阵的元素和是单调递增的。

我们可以计算二维前缀和。令 `sum[i][j]` 表示以 `(0, 0)` 为左上角，`(i, j)` 为右下角的子矩阵的元素和。
根据二维前缀和的性质，我们可以得到递推公式：
`sum[i][j] = sum[i-1][j] + sum[i][j-1] - sum[i-1][j-1] + grid[i][j]`

为了方便处理边界条件，我们通常会将 `sum` 数组的行和列都增加 1，即 `sum[i+1][j+1]` 表示以 `(i, j)` 为右下角的子矩阵的元素和：
`sum[i+1][j+1] = sum[i][j+1] + sum[i+1][j] - sum[i][j] + grid[i][j]`

在计算前缀和的过程中，如果发现某个 `sum[i+1][j+1] <= k`，则说明以 `(i, j)` 为右下角的子矩阵满足条件，我们将结果加一。如果 `sum[i+1][j+1] > k`，由于矩阵元素非负，继续向右或向下扩展的子矩阵元素和必然也会大于 `k`，但我们仍需计算完整的前缀和以便后续计算（不过在实际优化中，如果某一行在某一列开始大于 `k`，这一行后面的列都不用再统计了，甚至由于单调性可以进一步剪枝）。

这里给出标准的二维前缀和解法，时间复杂度为 $O(m \times n)$，空间复杂度为 $O(m \times n)$。由于我们只依赖上一行和当前行的前缀和，空间复杂度也可以优化到 $O(n)$。

### 代码实现

```python
class Solution:
    def countSubmatrices(self, grid: List[List[int]], k: int) -> int:
        m, n = len(grid), len(grid[0])
        # sum_mat[i+1][j+1] 表示以 (0,0) 为左上角，(i,j) 为右下角的子矩阵和
        sum_mat = [[0] * (n + 1) for _ in range(m + 1)]
        ans = 0
        
        for i in range(m):
            for j in range(n):
                sum_mat[i+1][j+1] = sum_mat[i][j+1] + sum_mat[i+1][j] - sum_mat[i][j] + grid[i][j]
                if sum_mat[i+1][j+1] <= k:
                    ans += 1
                else:
                    # 由于所有元素都是非负的，如果当前和已经大于 k，
                    # 那么同一行右边的元素也会大于 k，可以提前 break
                    break
                    
        return ans
```

```java
class Solution {
    public int countSubmatrices(int[][] grid, int k) {
        int m = grid.length, n = grid[0].length;
        int[][] sum = new int[m + 1][n + 1];
        int ans = 0;
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                sum[i+1][j+1] = sum[i][j+1] + sum[i+1][j] - sum[i][j] + grid[i][j];
                if (sum[i+1][j+1] <= k) {
                    ans++;
                } else {
                    break;
                }
            }
        }
        
        return ans;
    }
}
```

```cpp
class Solution {
public:
    int countSubmatrices(vector<vector<int>>& grid, int k) {
        int m = grid.size(), n = grid[0].size();
        vector<vector<int>> sum(m + 1, vector<int>(n + 1, 0));
        int ans = 0;
        
        for (int i = 0; i < m; ++i) {
            for (int j = 0; j < n; ++j) {
                sum[i+1][j+1] = sum[i][j+1] + sum[i+1][j] - sum[i][j] + grid[i][j];
                if (sum[i+1][j+1] <= k) {
                    ans++;
                } else {
                    break;
                }
            }
        }
        
        return ans;
    }
};
```

### 复杂度分析
- **时间复杂度**：$O(m \times n)$，其中 $m$ 和 $n$ 分别是矩阵 `grid` 的行数和列数。我们需要遍历整个矩阵计算二维前缀和。
- **空间复杂度**：$O(m \times n)$，用于存储二维前缀和数组。可以优化为 $O(n)$，只保留上一行和当前行的前缀和。
